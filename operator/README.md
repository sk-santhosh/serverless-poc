# QueueWorker Operator

Generic Kubernetes operator that watches `QueueWorker` custom resources and
creates single-shot `batch/v1` Jobs to drain a Redis Stream, scaling the
number of concurrent Jobs to queue depth (bounded by `spec.maxConcurrency`).
Nothing in this package is specific to sending email — see the top-level
[`../README.md`](../README.md) for how this operator is wired up to drive
the welcome-email worker.

## Layout

- [`api/v1alpha1/queueworker_types.go`](api/v1alpha1/queueworker_types.go) — the `QueueWorker` CRD's Go types.
- [`internal/controller/queueworker_controller.go`](internal/controller/queueworker_controller.go) — the reconcile loop.
- [`main.go`](main.go) — controller-runtime manager entrypoint.
- [`config/crd/queueworker-crd.yaml`](config/crd/queueworker-crd.yaml) — CRD manifest generated by `controller-gen`; this is the source of truth, copied verbatim into [`../deploy/helm/operator/crds/queueworker-crd.yaml`](../deploy/helm/operator/crds/queueworker-crd.yaml) for packaging.
- [`config/rbac/role.yaml`](config/rbac/role.yaml) — RBAC generated from the `+kubebuilder:rbac` markers in the controller; mirrored by hand into the Helm chart's `operator-rbac.yaml`.

## Regenerating CRD / RBAC / deepcopy

```bash
go install sigs.k8s.io/controller-tools/cmd/controller-gen@v0.16.5
export PATH="$PATH:$(go env GOPATH)/bin"

controller-gen object:headerFile="hack/boilerplate.go.txt" paths="./api/..."
controller-gen crd paths="./api/..." output:crd:dir=config/crd/bases
mv config/crd/bases/*.yaml config/crd/queueworker-crd.yaml && rmdir config/crd/bases
controller-gen rbac:roleName=queueworker-operator-role paths="./internal/controller/..." output:rbac:dir=config/rbac
```

After regenerating the CRD, copy it into `deploy/helm/operator/crds/queueworker-crd.yaml`.

## Assumptions / deviations from the spec

- **Go toolchain version**: the spec suggested `golang:1.22-alpine` for the
  Dockerfile's build stage. Current `k8s.io/client-go` and
  `sigs.k8s.io/controller-runtime` releases require a newer Go language
  version than 1.22 (resolved transitively by `go mod tidy`). The
  [`Dockerfile`](Dockerfile) build stage uses `golang:1.26-alpine` instead,
  matching the `go` directive in [`go.mod`](go.mod).
- **`main.go` location**: kept at the module root (classic kubebuilder v2/v3
  layout) rather than under `cmd/`, since the spec's repo tree didn't call
  out a `cmd/` directory.
- Go module path: `github.com/sk-santhosh/serverless-poc/operator` (not a
  real published module — this operator is not intended to be `go get`-able,
  only built into a container image).
